# `projects/` ディレクトリについて

このディレクトリにはサンプルや検証用のプロジェクトを配置します。  
新しいプロジェクトを追加する際は、以下のルールに従ってください。

## 追加手順

1. `projects/<project-name>` という形でディレクトリを作成する。  
2. プロジェクトの初期化を行う（例: Go の場合は `go mod init`）。  
3. `README.md` を用意し、セットアップ手順・実行方法・テスト方法を記載する。  
4. 最低限のテスト (`go test ./...` など) を追加し、緑にする。  
5. ルートの `autopilot.yml` の `project_dir` が追加したプロジェクトを指しているか確認し、必要なら更新する。

## 命名ルール

- ディレクトリ名は `kebab-case` で記述する（例: `sample-go-api`）。  
- できるだけプロジェクトの内容が伝わる短い名前にする。  
- 既存プロジェクト名との重複を避ける。
